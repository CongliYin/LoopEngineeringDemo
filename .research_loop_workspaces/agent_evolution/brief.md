# Research Brief

Topic: Agent 自主进化的技术路线、工程实现方式与评估方法

目标：
- 输出一份可用于字节跳动 agent 核心团队技术对齐的中文长报告。
- 读者看完后应能知道：近两年 Agent 自主进化有哪些关键技术、这些技术怎么实现、适合用在什么场景、工程上如何落地、如何评估是否有效。
- 不写短摘要式材料；不要把篇幅浪费在“摘要”和“关键结论”这类短报告结构上。
- 报告必须重视技术路线本身。`关键技术路线详解` 是全文主体，不能只占少量篇幅；每条路线都要让读者知道“这条线到底在做什么、代表工作怎么做、工程上怎么复现或借鉴”。
- 报告要有阅读欲望和清晰组织：避免一整片长段落堆叠，使用路线总览表、三级标题、路线卡片、Mermaid 图、代码片段和短小结来组织复杂内容。
- 引用来源不能只写成一句话带过。作为代表工作的来源，需要展开介绍其问题设定、核心方法、实验或评估方式、结论边界，以及它对工程落地的启发。
- 报告必须重视技术全景、技术脉络、实现细节、代码级设计、评估方案和落地选型，但不要让这些章节挤压关键技术路线的篇幅。
- 输出必须可审计：关键判断要有来源引用，工程推论要明确说明推导依据。
